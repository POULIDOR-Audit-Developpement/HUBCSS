# Security Policy

## Reporting A Vulnerability

Do not report security vulnerabilities in public issues.

Use GitHub private vulnerability reporting if it is enabled for the repository.
If that is not available, contact the maintainers through a private channel and
include:

1. A clear description of the issue.
2. Affected versions or commit ranges.
3. Reproduction steps or a proof of concept.
4. Any suggested mitigation.

## Response Goals

The project aims to acknowledge reports quickly, confirm impact, and publish a
fix or mitigation plan once the issue is verified.

## Scope Notes

This repository is a static design-system package. The main security concerns
are supply-chain integrity, accidental external runtime dependencies, and
unsafe distribution changes.
